# FindSubstringInString

Сравнение производительности алгоритмов поиска подстроки в строке — метода грубой силы (BF) и алгоритма Кнута–Морриса–Пратта (KMP). Тестирование проводится по методике измерения времени с доверительными интервалами.

## Алгоритмы

**Brute Force (BF)** — метод грубой силы. Для каждой позиции в тексте сравнивает символы с паттерном. Сложность: O(N·M) в худшем случае, O(N) в лучшем.

**Knuth–Morris–Pratt (KMP)** — использует префикс-функцию для пропуска заведомо неподходящих позиций. Сложность: O(N+M) во всех случаях.

Где N — длина текста, M — длина паттерна (в данном проекте M = 1000).

Весь пайплайн одной командой:

```bash

```

Или по шагам:


После выполнения в корне проекта появятся три графика:

- `graph_best_case.png` — лучший случай
- `graph_worst_case.png` — худший случай
- `graph_random_case.png` — случайные данные

## Методика измерений

Тестирование проводится по следующей схеме:

- **3 холостых запуска** перед замерами для прогрева кеша процессора
- **11 замеров** для каждой пары (алгоритм, данные)
- Итоговое время — среднее арифметическое замеров
- Погрешность — доверительный интервал по распределению Стьюдента (α = 95%, t = 2.2281)

На графиках экспериментальные точки не соединяются линиями; отдельно строится линия линейной аппроксимации O(N).

## Тестовые случаи

Для каждого размера данных генерируются три набора:

| Случай | Текст | Паттерн | Особенность |
|--------|-------|---------|-------------|
| Лучший | `"B" * N` | `"A" * M` | BF завершается после 1 сравнения на каждой позиции |
| Худший | `"A" * N` | `"A"*(M-1) + "B"` | BF делает M сравнений на каждой позиции |
| Случайный | случайные символы, паттерн вшит в хвост | случайный | реалистичный сценарий |

Размеры текста для лучшего и случайного случаев: от 1 КБ до 20 МБ (13 точек на логарифмической сетке). Для худшего случая диапазон ограничен до 32 КБ, так как BF имеет сложность O(N·M) и при больших N время выполнения становится неприемлемым в интерпретируемом Python.
